Locale module names in PUB 2.x+

Locale Names in PUB

ISO 639 (-1, -2, and -3) provide officially designated codes for a large number of languages. Please use these codes to identify PUB locale modules:
  • If an ISO 639 2-LETTER code exists, use it: English --> 'en'
  • If not, then use an ISO 639 3-LETTER code: Cherokee --> 'chr'
  • Otherwise, use "x_" plus a name for the language: Sindarin --> 'x_Sindarin'
Note that this does not provide for regional variations. Those are written using an underscore ("_"), followed by the region code in caps, like this: 
en_US  (United States English)
en_UK  (United Kingdom English)

PUB will treat the characters before the "_" specially -- if no module exists with the exact name requested by the LOCALE, then one sharing the same language code will be used.

Naming of PUB ENGINE locales

Once you know the correct locale code, you will create three files to localize the engine:
locale.py

Grammar

Grammar definition module. This is a python module which will declare the structural information needed to parse and generate the grammar of the language. It will also provide functions for correctly conjugating verbs and declining nouns, as well as maintaining any aesthetic agreements the language requires (such as euphonic spelling changes or gender or number matching). You can pretty much do whatever you want here, as there are hooks throughout the process.
locale.po

Messages

This is a gettext messages file. It contains specific messages from the engine which need to be translated. If you've ever worked on a gettext internationalization project, then this will already be familiar to you.
locale.xml

Vocabulary

This XML file defines the vocabulary to be understood by the parser. Words are glossed as closely as possible. But remember that you are dealing with an IF engine. Even in the original English, some of the meanings are stretched or simplified to cope with the IF world model.

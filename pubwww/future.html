Status and Future Plans for PUB

We had a lot of Ambitious plans at one time on this page, but I'm going to leave it open for now. If you are interested in where PyUniverse/PUB is going, please have a look at the Mailing List Archives where it has been/is being discussed (2004-2).

I plan to add a wiki site here to make this go a bit smoother, but I'm currently having technical problems with installing ZWiki. :-P, so please bear with me (2004-2).

Some ideas I still like:

  • XML room format -- it would be nice, for the novice user, for rapid prototyping, and for xcircuit conversion to create a parser that reads an XML room description file and generates a "dungeon" in PUB automatically. I've tinkered with Python's libxml and DOM interface, and I think I might find some time to do this (looks pretty easy). Should this be a real "Interactive Fiction Markup Language" as Jm originally called it? I don't know, but it might be useful. Expert users will continue to want the power of programming PUB in python, though.

  • Oz-like characters. I'm pretty sure this is possible, but it will require a lot of work!

  • Some form of graphical front-end integration. I think the Blender option probably represents the most impact for the least effort, but it does require familiarity with the Blender engine, which I don't have yet.
